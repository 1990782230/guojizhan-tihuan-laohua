# RGB 纹样替换与 AI 融合技术规范

版本：1.0  
状态：当前唯一有效流程  
用途：防止上下文压缩、模型切换或后续调试时改变已经确认的处理逻辑。

## 1. 最终目标

输入：

1. 一张带旧纹样的产品照片；
2. 五个新纹样元素。

输出：

1. 保持原产品结构、背景、光影和构图的最终图片；
2. 旧纹样全部清除；
3. 新纹样按照旧槽位类别一对一替换；
4. 次级纹样按包袋自身坐标从上到下、每行从左到右执行连续 ABAB；
5. 新纹样自然继承包身颗粒、光照、曲率、磨损和印刷质感。

## 2. 唯一坐标依据：RGB 标签图

后续所有步骤只使用同一张 RGB 标签图，不再次检测槽位，不建立第二套坐标。

颜色定义：

| RGB | 含义 |
|---|---|
| `0,0,0` | 非包身、非纹样区域 |
| `0,0,255` | 可编辑包身表面 |
| `255,0,0` | 原字母组合纹样 |
| `255,0,255` | 原菱形主纹样 |
| `255,128,0` | 原圆形纹样 |
| `0,255,0` | 原次级小纹样 |

规则：

- 纹样颜色优先于蓝色包身颜色；
- 黑色皮革、手柄、五金、包边、背景和其他物品保持黑色；
- RGB 图必须为 PNG；
- RGB 图与原图保持相同宽高比；
- 对齐原图时只使用最近邻插值；
- 不使用 JPEG、渐变、阴影、文字或抗锯齿颜色作为类别。

包身有效区域：

```text
蓝色区域 ∪ 所有纹样颜色区域
```

## 3. 固定三阶段流程

### 阶段一：RGB 图生成

输入：

```text
原始产品图
```

输出：

```text
01_rgb_label_raw.png
02_rgb_label_aligned.png
```

RGB 图一次性提供：

- 包身有效区域；
- 纹样类别；
- 纹样中心；
- 原槽位范围；
- 边缘裁切位置；
- 后续清除和替换所需坐标。

### 阶段二：由同一 RGB 图生成两个结果

#### A. 干净底图

```text
原图 + 所有纹样颜色形成的蒙版
→ AI清除旧纹样
→ 恢复连续包身材质
→ 03_clean_base.png
```

`03_clean_base.png` 直接采用 AI 返回的完整无纹样产品图。不得将清除结果按小纹样蒙版局部贴回原始带纹样照片，否则会重新带入旧纹样边缘、颜色残影和压印痕迹。

清除提示词：

```text
图1是产品原图。
只清除蒙版内原有纹样，恢复连续自然的无纹样包身表面。
保持原包身颜色、材质颗粒、光照、曲率和磨损。
不要增加任何纹样，保持产品结构、手柄、五金、包边、背景和构图不变。
```

#### B. 新纹样目标效果图

程序直接读取 RGB 连通区域：

```text
红色槽位 → 元素一
紫色槽位 → 元素三
橙色槽位 → 元素四
绿色槽位 → 按行优先顺序连续替换为 A、B、A、B……
```

输出：

```text
04_new_pattern_layer.png
05_target_preview.png
```

要求：

- 新纹样元素只使用透明 Alpha 形状；
- 从原图各 RGB 槽位采样旧纹样颜色；
- 采样时以槽位周围包身颜色为基准，选择与局部底材色差最大的原纹样像素，避免把包身底色误当成印刷色；
- 每个槽位独立继承对应旧纹样的局部颜色，以保留多色产品和局部明暗变化；
- 对采样结果只限制异常暗部与高光，不把所有槽位强制改成同一颜色；
- 程序同时计算包袋代表印刷色；单色产品出现孤立偏色时用代表色校正，检测到稳定多色分布时保留各槽位颜色；
- 禁止使用固定金色作为所有图片的默认配色，只有采样失败时才允许使用回退色；
- 不保留参考图中的黑色描边、底色或阴影；
- 新纹样层先与包身有效区域相交，包身外像素直接删除；
- `05_target_preview.png` 只负责位置、数量和排列，不追求最终融合。

### 阶段三：两图直接 AI 融合

只向最终图像模型提供：

```text
图1：03_clean_base.png
图2：05_target_preview.png
蒙版：完整包身有效区域
```

最终融合提示词：

```text
图1是无纹样的产品底图，图2是已经完成新纹样排列的目标效果图。
请以图1为基础，将图2中的新纹样自然融合到包身表面。
保持图2的纹样种类、数量、位置和排列，让纹样自然继承包身的材质颗粒、光照、曲面变化、磨损和印刷质感。
保持图2中已经从旧纹样采样得到的颜色，不得改成统一金色或其他配色。
保持包型、手柄、五金、包边、背景和构图不变。
```

最终直接采用 AI 返回图。

## 4. 明确取消的步骤

以下步骤已经通过测试确认会引入黑边、错位、双影或重新排版，不再使用：

- 第二套槽位检测；
- 逐纹样硬蒙版最终融合；
- 脚本模拟材质；
- AI 返回图逐纹样贴回；
- AI 返回图全局透视配准后局部合成；
- AI 无纹样结果按旧纹样小蒙版局部贴回原图；
- 最终结果再次叠加原纹样层；
- 将原始带旧纹样照片传给最后一次 AI 融合；
- 将带原品牌文字的原纹样参考传给最后一次 AI 融合。

## 5. 最终 AI 输入中允许和禁止的内容

允许：

- 无旧纹样干净底图；
- 已经放置新纹样的目标效果图；
- 包身有效区域蒙版。

禁止：

- 原始旧纹样图层；
- 旧纹样参考图；
- 原纹样蒙版的可视化版本；
- RGB 标签图；
- 检测框、编号和辅助线；
- Photoshop 图层截图。

RGB 标签图只服务于阶段一和阶段二，不传入最终融合调用。

## 7. 验收顺序

每次测试按以下顺序检查：

1. RGB 包身区域是否覆盖完整包身且不覆盖手柄、五金和背景；
2. 四种原纹样类别是否识别正确；
3. 干净底图是否清除全部旧纹样；
4. 新纹样层是否存在黑边；
5. 新纹样是否全部位于包身区域；
6. A/B 是否按行优先连续交替；
7. 目标效果图是否保持原构图；
8. 最终 AI 是否保留新纹样类型和排列；
9. 最终图是否存在黑色光晕、底部双影或原纹样恢复；
10. 记录每次 API 和本地处理耗时。

## 11. 非纹样线条保留规则

### 11.1 定义

包身上的以下连续结构不是待替换纹样：

- 直线、斜线和曲线；
- 连续格纹、交叉网格和连接线；
- 压印线、装饰线、绗缝线与车缝线；
- 贯穿多个纹样之间的连续线性结构。

这些结构必须保留，不参与元素一、三、四或 A/B 槽位分类。

### 11.2 RGB 标签阶段

RGB 标签模型必须将线条标为蓝色包身区域，而不是绿色次级纹样。只有独立、紧凑、具有封闭外轮廓的图形才可作为纹样槽位。

### 11.3 本地分离阶段

程序对绿色候选区执行两类线条检测：

1. 使用最小外接旋转矩形识别高长宽比连通区域；
2. 使用 Hough 线段检测连续直线或斜线。

检测遵守“紧凑闭合纹样优先”的通用不变量：

- 面积、短边、长宽比和填充率符合紧凑闭合图形的完整连通区域，先进入纹样保护区；
- Hough 线段只能从未受保护的区域提取线条；
- 最终组件过滤再次检查与纹样保护区的重叠，禁止因为一个大四瓣纹样内部存在长轴而把整个纹样判为线条；
- 只有高长宽比结构、低填充率长结构或由多个短线段组成的线性网络才进入线条蒙版。

输出：

```text
02_mask_preserved_lines.png
```

### 11.4 清除与融合阶段

1. 清除旧纹样时，提示词明确要求保留全部线条；
2. AI 返回干净底图后，再从原图确定性恢复检测到的线条像素；
3. 最终 AI 融合后再次恢复线条；
4. 恢复时排除新纹样透明层覆盖范围，避免线条错误绘制在新纹样上方。

由此形成：

```text
旧离散纹样：删除
连续包身线条：保留
新纹样：按槽位替换并自然融合
```

## 12. 其他批次修正

### 12.1 多面板包身

包身蒙版不再只保留面积最大的单一连通区，而是保留达到面积阈值的全部有效包身面板，适配正面、侧面和被结构件分隔的包身。

### 12.2 白底边界与底边裁切

- 使用白色背景前景掩码排除背景和中性阴影；
- 根据原图垂直梯度估计真实产品底边；
- 包身蒙版和新纹样统一裁切到产品轮廓内；
- 被边界截断的原槽位可以自然保留局部裁切，但不得延伸到白底或台面。

### 12.3 纹样配色继承

颜色不使用固定金色。配色继承与几何识别分离：

1. 比较原图与干净底图，获得旧印花残差色；
2. 同时统计 RGB 语义纹样区域内可见印花主体的直接颜色中值；
3. 当直接颜色与残差色在 Lab 空间一致时，优先采用直接颜色，避免最强残差只采到抗锯齿暗边；
4. 单色纹样使用统一源色，多色纹样保留逐槽位颜色；
5. 最终 AI 融合后，只在新纹样透明区域内校正 Lab 色度，不改变亮度和材质纹理。

由此分别约束：

```text
几何：纹样类型、槽位、尺寸、ABAB
颜色：源图旧纹样的色相与饱和度
融合：AI负责材质颗粒、光照和曲面
```

### 13.1 1169 批次结果纠正

相同输入出现了两次不同的 RGB 标签结果：

```text
线条专项运行：19个有效纹样槽位，线条分离正确
批次重跑：62个槽位，其中大量短缝线被误判为次级纹样
```

因此批次目录中的 1169 正式成品已改为线条专项运行的成品。被替换的错误批次成品保存在任务目录的 `_history` 中。

程序现增加两项防护：

1. RGB 标签提示明确规定：即使缝线由多个分离短线段组成，只要共同构成连续直线、斜线、曲线、格纹或绗缝线，全部标为蓝色，不得标绿；
2. 再次运行最终融合前，自动把现有 `07_final_ai.png` 保存到 `_history`，防止较差的新生成结果无痕覆盖已经确认的优质结果。

批次验收不能只检查 API 返回成功，还必须检查 RGB 标签中的槽位数量与线条分类是否合理。
