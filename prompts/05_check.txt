任务：对包袋印花替换结果进行严格复检，并只输出 JSON，不要使用 Markdown 代码块，不要输出其他文字。

你将收到图片：
- 如果共有三张：图1为替换前的包袋图，图2为已完成印花替换的结果图，图3为元素参考图。
- 如果共有两张：图1为已完成印花替换的结果图，图2为元素参考图。

检查图2（或两图模式下的图1）是否仍保留需要被替换的旧纹样或旧文字。重点检查：包身主面板、侧面、底部、包边附近的同材质区域、五金附近的皮革/涂层帆布、手柄、短肩带、长肩带、连接皮片、皮革挂饰、拉链皮尾；金属五金本身保持不动。

仅报告确实看得见的遗漏：旧字母组合、旧菱形主纹、旧圆形纹、旧小型次级花纹，以及 LOUIS VUITTON/PAIRS（不分大小写）文字遗漏。不要把金属纹理、缝线、阴影、皮革颗粒或模糊区域误判为遗漏。

元素参考图仅用于识别目标元素，不复制其文字、背景、分隔线或装饰。小型次级花纹继续遵守每个包袋由上到下、每行由左到右连续编号的 A-B-A-B 交错规则。

严格输出以下 JSON 结构：
{
  "needs_repair": true,
  "findings": ["简短描述每一处确实遗漏的位置和应替换的元素"],
  "repair_prompt": "一条可直接用于图片编辑的中文修复指令；只修复 findings 中的遗漏，保留其他全部内容不变，保持位置、大小、透视、遮挡、皮革纹理、光照和五金不变。"
}

如果没有确实遗漏，输出：
{
  "needs_repair": false,
  "findings": [],
  "repair_prompt": ""
}
